```mermaid
sequenceDiagram
    autonumber
    actor Player as 플레이어
    participant UI as 아이템획득_화면
    participant Battle as 전투
    participant Pl as 플레이어
    participant Ch as 캐릭터
    participant Inv as 인벤토리
    participant Item as 아이템

    Player->>+UI: 아이템획득 요청(플레이어id, 아이템명, 타입, 가치)
    UI->>+Battle: 아이템획득(플레이어id, 공격자, 아이템명, 타입, 가치)
    
    %% 플레이어 체크 (Include)
    Battle->>+Pl: 플레이어체크(플레이어id)
    Pl-->>-Battle: 검증 결과 반환 (boolean)
    
    %% 플레이어 체크 결과에 따른 최상단 분기
    alt 검증 결과 == true (체크 성공)
        %% 등급 판별 및 아이템 생성 로직
        Note over Battle: [아이템 등급 판별]<br/>가치 1000 이상: 전설 / 500 이상: 희귀 / 500 미만: 일반
        Battle->>+Item: 생성(아이템명, 타입, 가치, 등급)
        Item-->>-Battle: 신규 아이템 객체 반환
        
        %% Composition 관계에 따른 인벤토리 접근
        Battle->>+Ch: get인벤토리()
        Ch-->>-Battle: 인벤토리엔티티 반환
        
        %% 인벤토리 최대 용량 체크 및 아이템 추가 처리
        Battle->>+Inv: 아이템추가(신규아이템)
        Inv-->>-Battle: 추가 결과 반환 (boolean)
        
        %% 인벤토리 추가 결과에 따른 내부 분기
        alt 추가 결과 == true (용량 내)
            Note over Ch, Item: 캐릭터 "1" *-- "1" 인벤토리 "1" *-- "*" 아이템
            Battle-->>UI: 획득 성공 메시지 (String)
            UI-->>Player: 아이템을 획득했습니다.
        else 추가 결과 == false (최대 용량 10개 초과)
            Battle-->>UI: 획득 실패 메시지 (String)
            UI-->>Player: 인벤토리가 가득 차 아이템을 획득할 수 없습니다.
        end
        
    else 검증 결과 == false (체크 실패)
        Battle-->>UI: 검증 실패 메시지 (String)
        UI-->>Player: 올바르지 않은 플레이어 ID입니다.
    end
    
    deactivate Battle
    deactivate UI
